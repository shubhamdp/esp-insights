Read critical data from the RTC storage.

Parameters
[out]sizeNumber of bytes read
Returns
Pointer to the data on success, otherwise NULL
Note
It is mandatory to call rtc_store_critical_data_release_and_unlock() if rtc_store_critical_data_read_and_lock() is successful.
Please avoid adding ESP_DIAG_EVENT, error/warning logs using esp_log module in between rtc_store_critical_data_read_and_lock() and rtc_store_critical_data_release_and_unlock() API calls. It may lead to a deadlock.

◆ rtc_store_critical_data_release()

esp_err_t rtc_store_critical_data_release ( size_t  size)

Release the size bytes critical data from RTC storage.

This API can be used to remove data from buffer when data is sent asynchronously.

Consider data is read using rtc_store_critical_data_read_and_lock and sent to cloud asynchronously. Since status of data send is unknown, call rtc_store_critical_data_release_and_unlock with zero length. When acknowledgement for data send is received use this API with appropriate size to remove the data from the buffer.

Parameters
[in]sizeNumber of bytes to free.
Returns
ESP_OK on success, appropriate error code otherwise.

◆ rtc_store_critical_data_release_and_unlock()

esp_err_t rtc_store_critical_data_release_and_unlock ( size_t  size)

Release the utilized data read using rtc_store_critical_data_read_and_lock()

This API releases the utilized data read using rtc_store_critical_data_read_and_lock(). Utilization may involve encoding data, sending data to the cloud, post processing or printing on the console, etc.

Parameters
[in]sizeNumber of bytes to free. If all the data is utilized then pass the size returned by rtc_store_critical_data_read_and_lock() or 0 if no data is utilized (e.g. sending to cloud failed).
Returns
ESP_OK on success, appropriate error code otherwise.
Note
Please avoid adding ESP_DIAG_EVENT, error/warning logs using esp_log module in between rtc_store_critical_data_read_and_lock() and rtc_store_critical_data_release_and_unlock() API calls. It may lead to a deadlock.

Write non critical data to the RTC storage.

This API overwrites the data if non critical storage is full

Parameters
[in]dgData group of data eg: heap, wifi, ip(Must be the string stored in RODATA)
[in]dataPointer to non critical data
[in]lenLength of non critical data
Returns
ESP_OK on success, appropriate error code otherwise.
Note
Data is stored in Type-Length-Value format Type(Data group) - 4 byte - Pointer to the string in rodata Length - 4 byte - Length of data Value - Length byte - Data
